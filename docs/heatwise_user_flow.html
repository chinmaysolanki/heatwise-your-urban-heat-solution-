Screen Detail — Onboarding Screens
SCREEN ID SCREEN NAME PURPOSE NAVIGATES TO
splash Splash Screen Animated brand logo display on cold launch. Timed 3 s auto-advance or tap to skip. → onboarding
onboarding Onboarding Walkthrough 3-card horizontal carousel explaining core value props: Scan → Plan → Cool. Swipe or tap arrows. → phone auth (last card "Get Started")
auth/phone Phone Authentication User enters mobile number. System sends OTP via SMS (Twilio). Dev mode logs OTP to console. → otp verify
auth/otp OTP Verification 6-digit code entry. 30 s resend cooldown. Validates against hash+expiry in DB. Creates NextAuth session. → profileSetup (new user)
→ home (existing user)
profileSetup Profile Setup 2-step form: name (step 1) and city (step 2). Saves user record. Defaults: experience=intermediate, role=homeowner. → home

Screen Detail — Scan & Measurement Screens
SCREEN ID SCREEN NAME PURPOSE & KEY INTERACTIONS NAVIGATES TO
create Project Creation User names the project, selects primary goal (cooling / mixed), sets surface type. Creates a PhotoSession record. Existing projects shown as cards. → photoCapture
→ existing project (tap)
photoCapture Photo Capture Full-screen camera with framing guide overlay. User taps shutter to capture JPEG. Option to retake. Stored as base64 data URL on photoSession. → liveARMeasure
→ photoMeasureAR
→ measure (manual)
liveARMeasure Live AR Measurement WebGL Three.js live view with depth-sensing. User points camera at space edges. Real-time distance readout. Tap to confirm corners. Derives widthM × lengthM + floorLevel. → environment (confirm)
photoMeasureAR AR Photo Measurement Shows captured photo. User taps 2 opposing corners on screen. Pixel distance + known reference → metric dimensions. Confidence score displayed. → environment (confirm)
measure Manual Measurement Numeric inputs for width (m) and length (m). Floor level selector. Optional tape-measure guide. Proceeds to environment when both fields filled. → environment
environment Environment Setup Auto-detects location, temperature, UV index, wind speed via geolocation + weather API. User can override sun exposure, wind level, space type sliders. Shows live weather card when detected. → analysis

Garden Layout Screen — Plan Canvas
Canvas Elements
· Architectural top-down plan (HiDPI canvas)
· Perimeter border beds (14% wide on all walls)
· 4 raised centre beds (A/B/C/D) in 2×2 grid
· Plant symbols: concentric rings (canopy/trunk)
· Compass rose, dimension lines with arrowheads
· Scale bar at bottom of canvas
Zone Legend & Actions
· PLANTING ZONES panel with coloured dots
· Species per zone with count badges
· Tap zone → expanded detail panel
· "View Detailed Recommendations →" CTA
· "View Full Report" secondary CTA

AI Visualization — gpt-image-1 Pipeline
Input Signals to Prompt
· Space dimensions (widthM × lengthM m²)
· Floor level → elevation context
· Space type (rooftop / balcony / terrace)
· Environment (sun exposure, wind, temp °C)
· Scored plants (up to 12): type + visual descriptor
· Layout zones: positions as % of total space
· Infrastructure: planter module positions
· Heat reduction targets (coverage%, shade%, °C)
Output & Storage
· gpt-image-1 → 1024×1024 PNG (base64)
· Stored as VisualizationRecord in DB
· SourcePhotoRef (128 char ref)
· LayoutSchema + SpatialMapping JSON
· VisualizationPrompt (2000 chars)
· GenerationVersion counter per session
· Displayed auto on screen entry (once)
· Manual "Regenerate" triggers new call
